# 서비스 방향 재정의 및 사업 전략

## 현재 문제점

지금 구조 그대로 가면 **"한 번 써보고 끝나는 장난감 서비스"** 로 끝날 확률 높음.

| 문제 | 이유 |
|---|---|
| "성별 예측" | 신뢰성 문제 + 법/윤리 리스크 |
| "재미용" | 재방문 이유 없음 |
| "무료 + 광고" | 트래픽 없으면 의미 없음 |

> 핵심 질문: **"지속적으로 쓰게 만들 수 있냐?"**

---

## 방향 재정의

| 구분 | 내용 |
|---|---|
| ❌ 기존 | 초음파 → 성별 예측 |
| ✅ 개선 | 초음파 / 영상 → 상태 분석 + 재미 + 기록 서비스 |

---

## 확장 컨셉 (사업성 기준)

### 1. 태아 상태 AI 리포트

단순 성별 예측이 아닌 **의미 있는 정보** 제공

**기능:**
- 머리 / 팔 / 다리 위치 추정
- 움직임 분석 (활발함 / 안정)
- 자세 분석 (웅크림 / 펴짐)
- 주차별 특징 설명

**결과 예시:**
```
오늘의 아기 상태 리포트

- 움직임: 활발
- 자세: 몸을 둥글게 말고 있음
- 특징: 손을 얼굴 근처로 올림
```

→ **재미 + SNS 공유** 둘 다 가능

---

### 2. 성장 타임라인 (핵심 기능)

> 이거 없으면 망함

**기능:**
- 날짜별 초음파 기록
- AI가 변화 요약
- GIF / 영상 비교

**핵심 KPI:** 유저가 계속 업로드하게 만드느냐

---

### 3. 영상 분석 (차별화 포인트)

> 이거 들어가면 서비스 급 올라감

**기능:**
- 영상에서 움직임 패턴 추출
- "손 흔드는 것 같아요" 같은 자연어 해석

**경쟁력:** 대부분 이미지 기반 → 영상 해석은 희귀

---

### 4. 부모용 감성 기능 (공유 핵심)

> 이거 있어야 바이럴됨

**기능:**
- 자동 감성 문구 생성
- SNS 공유 카드 생성

**예시:**
```
우리 아기가 오늘은 많이 움직였어요 👶
```

→ **바이럴 요소**

---

### 5. AI 질문 시스템

> 붙이면 체류시간 증가

**기능:**
- "이거 정상인가요?"
- "왜 이렇게 보이나요?"

→ GPT API 연동으로 바로 구현 가능

---

## 수익 모델

### 1. 프리미엄 리포트
- 기본 무료
- 상세 분석 유료 (월 3,000 ~ 5,000원)

### 2. 병원 / 산부인과 B2B
- SaaS 형태로 납품
- 병원이 부모에게 제공하는 서비스로 활용

### 3. 광고
- 육아 / 유아용품 타겟 광고
- 부모층 → CPM 높음

---

## 리스크

### 의료법 / 윤리
- "진단"이라고 표현하면 위험
- 반드시 면책 문구 포함:
  > "본 서비스는 참고용 정보이며 의료적 판단이 아닙니다."

### AI 정확도
- 성별 예측 → 특히 위험
- 대신 **"패턴 기반 추정"** 으로 표현

## 확장 설계 - 현재 구조에서 추가 필요 항목

| 항목 | 현황 | 필요 작업 |
|---|---|---|
| 파일 저장 | 메타데이터만 저장 | S3 또는 로컬 스토리지 연동 |
| 분석 큐 | 없음 | Redis Queue 도입 (영상 처리 지연 대응) |
| 비동기 상태 | 없음 | "분석중 / 완료" 상태 필드 추가 |

## MVP 로드맵 (현실적)

> 처음부터 다 하지 말 것

### STEP 1
- 이미지 업로드
- 랜덤값 제거 → GPT API 기반 상태 텍스트 생성

### STEP 2
- 히스토리 저장 (성장 타임라인)

### STEP 3
- 영상 분석 도입
